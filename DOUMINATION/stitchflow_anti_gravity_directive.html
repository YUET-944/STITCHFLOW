02 Anti-Gravity Feature Logic CORE REQUIREMENTS
AG-001 Dual-Entry Gateway & Role Architecture CRITICAL
Landing screen renders two primary CTAs: Join as Tailor and Join as Client. Role is stored as a JWT claim (role: "TAILOR" | "CLIENT" | "ADMIN") and never re-selectable post-verification. A single Flutter widget tree uses a RoleRouter provider that injects either the TailorShell or ClientShell navigation scaffold — shared codebase, divergent UX. Supabase Row Level Security (RLS) policies enforce role isolation at the database layer independently of the API.
AG-002 Measurement Vault — Asymmetric Access Control CRITICAL
The measurements table is guarded by a triple gate: (1) JWT RBAC check at API middleware, (2) Supabase RLS policy: USING (tailor_id = auth.uid() AND EXISTS (SELECT 1 FROM tailor_client_links WHERE tailor_id = auth.uid() AND client_id = measurements.client_id)), (3) client app renders the vault as read-only (MeasurementVaultReadOnlyWidget). Writes require an active tailor_client_link record. Every write creates a new versioned row — no in-place updates. The is_current flag flips via a DB trigger.
AG-003 Searchable ID System — TR/CL Alphanumeric Keys CRITICAL
On registration confirmation, the backend generates a human-readable ID using: PREFIX-CITY_CODE-ZERO_PADDED_SEQUENCE. Example: TR-Mardan-001, CL-Karachi-042. The prefix is role-derived. City code is extracted from the Google Maps Place result (locality component, max 8 chars, uppercased). Sequence is an auto-increment from a id_sequences table partitioned by role+city. This ID is the primary search key on the Client's Track Order screen — a single input that fires a debounced API call returning the live delivery checklist.

// ID generation pseudocode (TypeScript)
const city = extractLocality(googleMapsResult); // "Mardan"
const seq = await db.nextVal(`seq_${role}_${city}`);
const id = `${role==='TAILOR'?'TR':'CL'}-${city}-${seq.toString().padStart(3,'0')}`;
AG-004 Capacity Gate & Booking Approval Workflow HIGH
Tailor profile stores max_active_orders (integer, 1–50) and current_active_orders (maintained by DB trigger on order status change). Before a booking is created, the BookingService.createBooking() method runs a serialised check inside a PostgreSQL transaction: SELECT current_active_orders FROM tailor_profiles WHERE id = $tailorId FOR UPDATE — if current >= max, throw CAPACITY_EXCEEDED (HTTP 409). If under capacity, the order is inserted with status PENDING. Tailor receives FCM push and in-app queue entry. Approval sets status CONFIRMED and increments the counter. Rejection requires a reason code from a predefined enum.
AG-005 Live Delivery Checklist — State Machine HIGH
Order status is an append-only state machine. Valid transitions (Tailor-only):
PENDING→ CONFIRMED→ CUTTING→ STITCHING→ QUALITY CHECK→ READY
Each transition is a PATCH /orders/:id/stage call that appends to stage_history[] (immutable audit log). Client gets FCM push + Supabase Realtime update. The Flutter DeliveryTracker widget uses a StreamBuilder on the Supabase subscription — zero polling, zero page refresh.
AG-006 Communication Layer — Call Button + Real-Time Chat CORE
One-Tap Call: Tailor phone (E.164 format) stored in users.phone. Visible to Clients only after a confirmed booking exists. Flutter: url_launcher package fires tel:{phone}. Web: renders as a clipboard-copy button.

Real-Time Chat: Firebase Realtime Database node: /chats/{chatChannelId}/messages/{pushId}. Channel auto-created on booking confirmation. Flutter uses firebase_database SDK with StreamBuilder for zero-latency message rendering. Supports: TEXT, IMAGE (Cloudinary upload → URL sent), SYSTEM (auto-generated on stage transitions). FCM dispatched via Cloud Function trigger on new message when recipient is backgrounded.
AG-007 Portfolio Gallery & Dynamic Pricing Display CORE
Tailors upload portfolio images via a direct Cloudinary upload widget (signed preset — server issues a short-lived signature, client POSTs directly to Cloudinary CDN, URL returned and stored in tailor_profiles.portfolio_urls[] — max 50 items). Gallery renders as a SliverGrid in Flutter (3-column on mobile, 4-column on web). Tap opens a full-screen PageView with pinch-to-zoom.

Pricing: price_per_suit_min / price_per_suit_max displayed as a range badge on the public profile. Editable only in the Tailor dashboard. Currency: PKR (v1.0). Multi-currency is a v2.0 feature flag.
